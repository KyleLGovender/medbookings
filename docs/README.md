## Quick Start

For new developers joining the project:

1. **Setup**: Follow the README.md in the root directory for local development setup
2. **Architecture**: Read the [Architecture Overview](./architecture.md) to understand the system design
3. **Components**: Browse [UI Components](./ui-components.md) to understand available building blocks
4. **Features**: Explore [Feature Components](./feature-components.md) for business logic components
5. **API**: Review [API Routes](./api-routes.md) for backend integration

## Documentation Guidelines

- All components should be documented with their props, usage examples, and any special considerations
- API routes include request/response schemas, authentication requirements, and error handling
- Hooks document their parameters, return values, and usage patterns
- Utilities include function signatures, parameters, and practical examples

## Contributing

When adding new components, hooks, or utilities:
1. Follow the existing code patterns
2. Add comprehensive JSDoc comments
3. Update the relevant documentation files
4. Include basic usage examples
5. Update this index if adding new documentation sections
